# 需求：Doctor CRM - 自費醫療管理系統

**定義日期：** 2026-03-26
**核心價值：** 為醫療機構提供統一的自費診療管理平台，確保療程透明、財務清晰、患者體驗優質

## v1 需求

### 療程與醫令管理

- [x] **COURSE-01**：醫護人員能創建新療程（名稱、類型、費用、療程數）
- [x] **COURSE-02**：醫護人員能編輯療程詳情（目標、描述、進度等）
- [x] **COURSE-03**：系統能追蹤療程進度（已完成課程數/總課程數）
- [x] **COURSE-04**：系統能分配醫護人員到療程課程
- [x] **COURSE-05**：患者能查看自己的療程列表與進度

- [x] **SCRIPT-01**：醫師能創建醫令（藥物/治療內容、劑量、使用方式）
- [x] **SCRIPT-02**：系統能追蹤醫令使用狀態（未開始、進行中、已完成）
- [x] **SCRIPT-03**：患者能查看已開立的醫令與使用進度

### 患者進度提醒通知

- [x] **NOTIF-01**：系統能在療程開始時發送患者通知
- [x] **NOTIF-02**：系統能在療程進度變化時發送提醒（每次課程完成後）
- [x] **NOTIF-03**：系統能在療程完成時發送結果通知
- [ ] **NOTIF-04**：系統能發送預約時間提醒
- [ ] **NOTIF-05**：支援多渠道通知（郵件、簡訊、應用內通知）

### 財務與費用管理

- [ ] **FIN-01**：系統能準確計算療程總費用（課程費 × 課程數）
- [ ] **FIN-02**：系統能記錄患者支付記錄與餘額
- [ ] **FIN-03**：系統能生成費用明細（按療程、按日期）
- [ ] **FIN-04**：醫護人員能生成患者發票
- [ ] **FIN-05**：系統能支援多種支付方式（現金、轉帳、刷卡）
- [ ] **FIN-06**：系統能生成收入報表（按醫護人員、按時間段）

### 患者基本資料

- [x] **PATIENT-01**：系統能以身份證ID + 姓名唯一識別患者
- [x] **PATIENT-02**：系統能保存患者基本資料（聯絡方式、病史、過敏史等）
- [x] **PATIENT-03**：患者資料支援快速搜尋與篩選

## v2 需求

### Doctor Toolbox 整合

- **INTEGRATION-01**：API 對接設計文檔
- **INTEGRATION-02**：雙向資料同步機制
- **INTEGRATION-03**：身份證ID與姓名索引查詢
- **INTEGRATION-04**：共用資料庫架構

### 進階通知功能

- **NOTIF-ADV-01**：患者能自訂通知偏好
- **NOTIF-ADV-02**：患者能查看通知歷史
- **NOTIF-ADV-03**：支援預約時間前N天提醒

### 報表與分析

- **REPORT-01**：收入趨勢分析
- **REPORT-02**：療程完成率分析
- **REPORT-03**：患者滿意度評分

## 外部範疇

| 功能 | 原因 |
|------|------|
| 患者線上支付 | 涉及第三方金流，先用傳統記錄方式，未來版本再整合 |
| 患者自助線上預約 | 優先完成內部預約管理，後期考慮患者端功能 |
| 診療視訊遠距會診 | 複雜度高，超出 v1 範疇 |
| 醫療影像管理 | 待建構獨立模組 |
