   - Method: async executeWithRetry<T>(fn: () => Promise<T>, maxAttempts: number = 4): Promise<T>
     * maxAttempts default = 4 (try 1 + 3 retries per CONTEXT.md "3-5 attempts")
     * Backoff delays (ms): [2000, 4000, 8000, 16000] for attempts 1-4
     * Attempt 1 (delay 0): Initial try
     * Attempt 2 (delay 2000ms): First retry
     * Attempt 3 (delay 4000ms): Second retry
     * Attempt 4 (delay 8000ms): Third retry
     * Attempt 5 (delay 16000ms): Fourth retry if maxAttempts=5
   - Logic:
     * for (let attempt = 0; attempt < maxAttempts; attempt++) {
       - try { return await fn(); } catch (error)
       - if (attempt < maxAttempts - 1) { await delay(backoffDelays[attempt]); }
       - else { throw error; }
     }
   - Helper: private delay(ms: number): Promise<void> using setTimeout
   - Logging: Log each retry at warn level with attempt number and delay
   - Type-safe: Generic <T> for return type
   - No external dependencies (use Node.js setTimeout)

1. Create SyncPatientService at `backend/src/doctor-toolbox-sync/services/sync-patient.service.ts`:
   - Decorator: @Injectable()
   - Inject: PatientService, SyncIndexService, RetryService, ConfigService, Logger
   - Core methods:

   a) async syncFromToolbox(payload: WebhookPayloadDto, clinicId: string): Promise<Patient>
      - Extract Doctor Toolbox patient: const tbPatient = {id, idNumber, name, phone, email}
      - Exact lookup by (clinicId, idNumber, name): await syncIndexService.findByIds(clinicId, payload.patient.idNumber)
      - If found and crmPatientId exists: Apply update (CRM override) → jump to step 3
      - If not found: Fallback lookup (name + phone): await patientService.findOne({clinicId, name, phone})
      - If fallback found: Detect conflict (same person, different idNumber?) → merge
      - If new patient: Create in CRM with Toolbox data → jump to step 3
      - Return final patient entity

   b) async detectConflict(crmPatient: Patient, toolboxData: WebhookPayloadDto['patient']): Promise<boolean>
      - Conflict = same person, different/missing idNumber
      - Return true if crmPatient.idNumber !== toolboxData.idNumber && toolboxData.idNumber non-empty

   c) async mergePatients(crmPatient: Patient, toolboxData: WebhookPayloadDto['patient']): Promise<Patient>
      - CRM is authoritative: Keep crmPatient identity
      - Update optional fields only (phone, email) from Toolbox if CRM values empty
      - Never overwrite idNumber (CRM-controlled per CONTEXT.md)
      - Return updated patient

   d) async pushPatientToToolbox(patient: Patient, clinicId: string): Promise<void>
      - Transform CRM patient to Toolbox schema: {id, idNumber, name, phone, email}
      - GET config DOCTOR_TOOLBOX_WEBHOOK_URL and API key
      - POST to Toolbox with retry: await retryService.executeWithRetry(async () => {
          return await fetch(url, {method: 'POST', body: JSON.stringify(patient), headers: {...}})
        }, 5)
      - On success: Update syncStatus='synced' in sync_patient_index
      - On failure: Update syncStatus='failed' after all retries exhausted

   e) async findPatientExact(clinicId: string, idNumber: string, name: string): Promise<Patient | null>
      - Query by (clinicId, idNumber, name) exact match
      - Used by sync endpoint and listener

   f) async findPatientFallback(clinicId: string, name: string, phone: string): Promise<Patient | null>
      - Fallback query: (clinicId, name, phone)
      - Used if exact match fails

   - Error handling:
     * BusinessRuleException for data integrity issues
     * On Toolbox API error: Log and mark syncStatus='failed', don't throw
   - All methods include clinicId for multi-clinic isolation
