   - Title: Doctor CRM ↔ Doctor Toolbox API 整合文件
   - Sections:
     * 概述（Purpose, Scope, Authentication）
     * Webhook Endpoint：
       - POST /sync/webhook
       - Headers: x-signature (HMAC-SHA256), x-timestamp
       - Request body: WebhookPayloadDto schema with examples
       - Response: 200 {message, status}, 401 {error}, 400 {validation error}
       - Retry policy: 3-5 attempts with exponential backoff
     * Patient Lookup API（Internal）：
       - GET /patients/lookup?idNumber={id}&name={name}&phone={phone}
       - Exact match (idNumber + name), fallback (name + phone)
     * Audit Log API：
       - GET /sync/audit/logs/:patientId
       - GET /sync/audit/clinic
       - GET /sync/audit/stats
     * Error Codes：
       - 400: Invalid payload (missing fields)
       - 401: Invalid signature or timestamp
       - 409: Conflict detected (auto-merge applied)
       - 503: Sync service temporarily unavailable (retry)
     * Data Mapping：
       - Toolbox → CRM: {id → toolboxPatientId, idNumber, name, phone, email}
       - CRM → Toolbox: {id, idNumber (read-only), name, phone, email}
