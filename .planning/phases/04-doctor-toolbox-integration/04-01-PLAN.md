<task type="auto">
  <name>Task 3: Create WebhookSignatureGuard with HMAC-SHA256 validation</name>
  <files>
    backend/src/doctor-toolbox-sync/guards/webhook-signature.guard.ts
  </files>
  <action>
實作 Webhook 簽名驗證衛士，使用 HMAC-SHA256 確保 Doctor Toolbox 請求真實性。

實作要點（基於 RESEARCH.md Pattern 1）：
1. 讀取 x-signature 和 x-timestamp 請求頭
2. 時戳驗證：檢查 |now - timestamp| < 300 秒（防重放攻擊）
3. 簽名驗證：
   - 訊息格式：`${timestamp}.${JSON.stringify(body)}`
   - 使用 crypto.createHmac('sha256', secret) 生成簽名
   - 使用 crypto.timingSafeEqual() 比較（防時序攻擊）
4. 無效簽名 → 拋出 UnauthorizedException（HTTP 401）
5. 過期時戳 → 拋出 UnauthorizedException（HTTP 401）
6. 取得 secret 自 ConfigService（環境變數：DOCTOR_TOOLBOX_WEBHOOK_SECRET）

衛士應返回 true 表示驗證成功，Controller 可直接使用經過驗證的 request.body。

遵循現有衛士模式（見 backend/src/auth/guards/ 目錄）。
  </action>
  <verify>

npm run test -- webhook-signature.guard
curl -X POST http://localhost:3000/api/doctor-toolbox-sync/webhook -H "x-signature: invalid" -H "x-timestamp: $(date +%s)" -d '{}' && echo "Should fail" || echo "Guard working"
  </verify>
  <done>
Guard validates HMAC-SHA256 signature and timestamp, timingSafeEqual prevents timing attacks, rejects invalid signatures with 401
